# Husk – fuld audit 2026-06-21

Kørt med fire parallelle agenter (sikkerhed, robusthed, kode/F-Droid, UX/docs) på `co.xplat.husk` (v0.8)
+ xplat husk-siderne. Fund nedenfor med status: **[RETTET v0.9]** = ordnet i denne release, **[UDSKUDT]** =
dokumenteret til en senere runde (større/forstyrrende ændring).

## Kritisk / Høj

1. **0.0.0.0-modsigelse i docs.** Koden binder nu `0.0.0.0` + kilde-IP-ACL (v0.8), men fastlane-beskrivelser
   (en+da), README og manifest-kommentaren påstod stadig "binder kun loopback+Tailscale, aldrig 0.0.0.0".
   F-Droid-blocker + uærligt. **[RETTET v0.9]** – docs opdateret til at beskrive 0.0.0.0 + ACL korrekt.
2. **AdbForward (5557) havde præcis den binding-svaghed v0.8 fiksede i ControlServer** – bandt kun den
   enumererede Tailscale-IP og gen-bandt aldrig → scrcpy går permanent mørk ved IP-skift/reconnect.
   **[RETTET v0.9]** – binder nu `0.0.0.0` + delt `Net.peerAllowed`-ACL.
3. **Konverterings-fejl fra v0.7-digraf-ombytningen** (UX-agent): "kameraet"→"kameræt", "kommandoen"→
   "kommandøn" (morfem-grænser fejlkonverteret), + all-caps "SKAERM"/"FOERSTE" blev sprunget over.
   **[RETTET v0.9]** – rettet i app.py + ScreenService/ControlServer.
4. **Selv-opdatering uden APK-signatur/hash-tjek** (sikkerhed). Android håndhæver dog samme-nøgle ved
   opdatering, så praktisk RCE-risiko er begrænset. **[UDSKUDT]** – tilføj SHA-256 i latest.json + verificér.
5. **Token er valgfri (default tom)** → kun ACL beskytter; ACL admitterer hele LAN + hele tailnet.
   **[UDSKUDT]** – gøre token obligatorisk er forstyrrende for fleet/companion; dokumenteret som stærk
   anbefaling. Delvist afhjulpet: ACL strammet (link-local fjernet) + konstant-tids token-sammenligning.
6. **REQUEST_INSTALL_PACKAGES + indbygget selv-opdater** → F-Droid anti-feature. **[UDSKUDT]** – kræver
   build-flavor der ekskluderer updateren fra F-Droid-bygget; større refaktor, planlagt til næste runde.
7. **`typeAllMask` i accessibilityconfig** er bredere end nødvendigt (onAccessibilityEvent er tom).
   **[RETTET v0.9]** – indsnævret til `typeWindowStateChanged`.

## Medium

8. **Kamera reopener ikke ved onDisconnected/onError** → feed fryser men enheden ser nåbar ud.
   **[RETTET v0.9]** – reopen-watchdog med backoff.
9. **Batteri-optimering kun manuel** (ingen status/check); Samsung-deep-sleep dræbte A10e.
   **[RETTET v0.9]** – `isIgnoringBatteryOptimizations` eksponeret i /flags. (Samsung "sovende apps" skal
   stadig sættes manuelt – dokumenteret.)
10. **Ulokaliseret notifikations-tekst** (altid dansk, + "Skaermdeling" ae-fejl). **[RETTET v0.9]** – bruger
    `R.string.notif_text`.
11. **Android-12 FGS-baggrundsstart** kan kaste i InstallReceiver/BootReceiver (kamera-FGS). **[RETTET v0.9]**
    – fejlen logges nu (ikke tavst slugt) så /flags kan afsløre mislykket recovery; batteri-exempt løfter den.
12. **MediaProjection.Callback ikke registreret** (Android 14-risiko). **[RETTET v0.9]** – registreret.
13. Seq-tællere ikke-atomare (én skriver i praksis), /set-rotation slår ikke igennem live, redirect uden
    dybde-grænse, install læser hele APK i RAM, MJPEG-tråde reapes sent. **[UDSKUDT]** – mindre, noteret.

## Bekræftet OK (ingen handling)
Ingen secrets i repo; SPDX-headere i alle filer; dansk æ/ø/å korrekt i strings.xml + fastlane; ingen
 over-26-API uden guard (crasher ikke på Android 8); feature-gating + status-sandhed korrekt; reproducerbar
build-config; OpenAPI internt konsistent (37 endpoints); lys-tema-CSS uden sorte bokse; API-siden lækker
ingen rigtige IP'er/tokens.

## Verdikt
v0.8 var arkitektonisk sund men efterlod AdbForward-bindingen + doc-modsigelsen. v0.9 retter de kritiske
+ høj-værdi-sikre fund og tilføjer killer-feature'en. Tilbage til F-Droid-merge: build-flavor uden
selv-opdater (#6) + evt. obligatorisk token (#5).
